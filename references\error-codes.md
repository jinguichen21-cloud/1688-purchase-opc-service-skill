# Error Handling

## 常见问题

### 1. 未返回商品

表现：
- `offers` 为空
- 只有空对象或关键字段缺失

处理：
- 告诉用户当前未找到合适商品
- 建议补充采购品类、预算、规格、数量、品牌偏好

### 2. 下单链接缺失

表现：
- `landingPageUrl` 为空或不存在

处理：
- 可以先返回商品推荐
- 说明采购助手下单链接暂时不可用，不要自己猜测或手工拼接未知参数

### 3. 下单链接含 `${corpId}` 占位符

处理：
- 先读取当前悟空上下文真实 `corpId`
- 用真实值替换所有 `${corpId}`
- 如果当前没有企业上下文，不输出下单链接，并说明原因
- 页面输出时仍需包装成 `[1688企业采购agent](...)`

### 4. 商品描述过长

处理：
- 不要原样粘贴大段 `purchaseName`
- 提炼标题、1 到 2 个卖点、价格和链接即可

### 5. 页面只显示裸链接

处理：
- 改成 markdown 展示 `[1688企业采购agent](https://...)`
- 不要丢掉 `https://www.1688.com/link?url=` 前缀

### 6. 误触发下单场景

表现：
- 用户要求付款、查物流、售后、库存

处理：
- 不调用本技能
- 明确当前能力只负责推荐商品和引导进入采购助手
