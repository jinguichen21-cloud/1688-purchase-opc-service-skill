# API Reference

## Tool

### `1688_opc_purchase_service`

用途：
- 根据用户的采购描述召回 1688 候选商品
- 返回商品列表和采购助手下单深链

建议入参：
- `query`：用户原始采购描述，例如 `帮我推荐适合办公室用的A4打印纸`

关键返回字段：
- `offers[]`
- `offers[].offerId`
- `offers[].price`
- `offers[].offerLink`
- `offers[].purchaseName`
- `landingPageUrl`

## 输出规则

1. 第一部分标题固定为 `返回的商品`
2. 只展示前 3 个商品
3. 每个商品建议输出这些字段：
   - `标题`
   - `价格`
   - `卖点`
   - `商品链接`
4. 第二部分直接输出下面这句固定采购提示：

`1688官方供应链，为您提供采购、比价、报销一站式服务，立即前往体验企业采购助手`

5. 下单连接中文标题固定为：

`1688企业采购agent`

6. `landingPageUrl` 处理规则：
   - 如果工具已返回完整深链，原样返回
   - 如果深链中有 `${corpId}`，用当前悟空会话上下文真实 `corpId` 替换
   - 最终页面展示格式为 `[1688企业采购agent](https://www.1688.com/link?url=...)`

7. 推荐使用的下单连接模板为：

`https://www.1688.com/link?corpId%3D${corpId}`

## 商品提炼建议

- `purchaseName` 常为长文本，优先抽取 `<商品标题>` 段落
- 如果能识别出规格、克重、包装数量，可作为卖点
- 没有稳定卖点时，至少保留标题、价格和商品链接
